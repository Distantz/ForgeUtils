# Custom Tracked Ride (CTR) Overview

## Introduction
A Custom Tracked Ride (commonly known as CTR) mod provides a replacement of or an additional new tracked ride to the game. You can achieve replacements with prefab injection or overriding assets. Additions require new files to be created and referenced by the game's database. The purpose of this page is to show the general layout of a CTR in a mod. Specifics will be focused on in dedicated pages, which will be under this general header.

## Anatomy
### Concepts
A CTR mod generally contains these following concepts:
- Code:
    - `database.*MODNAME*luadatabase.lua` for a mod entry point
    - Prefabs:
        - `Track_*TRACKEDRIDEID*.lua`, which defines the prefab for the track
        - `.lua`, any other prefabs
- Database:
    - `.fdb`, for registering the CTR with the game. This is no longer recommended, you should either use the SQL Bindings or Builders present in ForgeUtils to add your CTR instead
- File formats:
    - Functionality:
        - `.assetpkg`, for loading of ovls
        - `.ppuipkg`, for defining UI-used files
        - `.trackelement`, for adding elements
        - `.trackstation`, for adding stations
        - `.trackedridecar`, for adding cars in a train
    - Visuals:
        - `.trackmesh`, for adding a visual representation of an element
        - `.supportset`, for adding custom autogenerated supports
        - `.ms2`, for adding 3D models and physics
        - `.fgm`, for defining visual material properties
        - `.tex`, for defining texture info, like DirectX compression, resolution and mipmaps
        - `.dds`, DirectDraw Surface (image) file
        - `.png`, Portable Network Graphics (image) file
- Translation files:
    - `.txt`, for naming other things in the mod. Any user-facing text is usually a `.txt` file

### Organisation
Mod files should be organised in the parent mod folder (the folder containing the `Manifest.xml`) like so:

- `/Init/`
    - `.txt`
    - `.assetpkg`
- `/Main/`
    - `.lua`
    - `.ppuipkg`
    - `.trackelement`
    - `.trackedridecar`
    - `/uigameface/img/trackedrides/`
        - `.dds`
        - `.png`
        - `.tex`
- `/Rides/*Type*/`
    - `/Cars/*ID*/`
        - `.ms2`
        - `.fgm`
        - `.tex`
        - `.dds`
        - `.png`
    - `/Tracks/*ID*/`
        - `.trackstation`
        - `.trackmesh`
        - `.supportset`
        - `.ms2`
        - `.fgm`
        - `.tex`
        - `.dds`
        - `.png`
